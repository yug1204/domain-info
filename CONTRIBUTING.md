# Contributing to DomainRecon

First off, thank you for considering contributing to DomainRecon! It's people like you that make DomainRecon such a great tool for the community.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if the bug has already been reported.
* If not, open a new issue. Clearly describe the bug, including steps to reproduce it and expected behavior.

### Suggesting Enhancements
* Open a new issue with a clear title and description of the suggested enhancement.
* Explain why this enhancement would be useful to other users.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bug fix: `git checkout -b feature-name`.
3. Make your changes and commit them with descriptive messages.
4. Push your changes to your fork.
5. Submit a pull request to the main repository.

## Development Setup

1. Install Python 3.10+.
2. Clone the repo: `git clone https://github.com/yug1204/domain-info.git`
3. Create a virtual environment: `python -m venv .venv`
4. Activate the virtual environment:
   - Windows: `.venv\Scripts\activate`
   - Linux/macOS: `source .venv/bin/activate`
5. Install dependencies: `pip install -r requirements.txt`
6. Run the dev server: `uvicorn main:app --reload`

## Code Style
* Keep the code clean and well-documented.
* Use meaningful variable and function names.
* Follow PEP 8 guidelines for Python code.

## Questions?
Feel free to open an issue for any questions or reach out to the maintainers.

Happy hacking! 🕵️‍♂️
